# 演示与运行手册

## 启动

1. 首次运行 04_配置AI密钥.cmd。
2. 运行 01_启动作品.cmd。
3. 前端：http://127.0.0.1:3003
4. 后端文档：http://127.0.0.1:8003/docs
5. 健康检查：http://127.0.0.1:8003/api/health

## 完整演示路径

1. 注册一个新的本地账号，证明数据隔离。
2. 记录三条梦境碎片，至少包含人物、地点和情绪。
3. 选择情绪强度并提交织梦。
4. 先查看结构化叙事，再等待场景图渐进生成。
5. 进入沉浸页逐幕回看。
6. 完成长按收梦仪式。
7. 打开梦境库、日历和详情。
8. 重新编织一次，确认旧资源被替换。
9. 重启项目，确认 SQLite 和本地图像仍存在。

## 后端检查

健康接口中的 provider 应为 stepfun，ai_enabled 应为 true。若为 fallback，先重新配置密钥。场景图应保存到项目源码/xunmeng-dream-agent/backend/data/assets。

## 关闭与排查

- 运行 02_关闭作品.cmd。
- 运行 03_检查状态.cmd，确认 3003 和 8003 已释放。
- 后端失败查看 .runtime/backend.stderr.log。
- 前端失败查看 .runtime/frontend.stderr.log。
- 模型失败检查 StepFun 套餐、API Base 和模型可用性。
